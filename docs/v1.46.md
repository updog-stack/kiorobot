## 개요
로그인 세션이 필요한 마케팅(당근·네이버 블로그) 수집을 **손이 덜 가게** 만들었습니다. 데이터 동기화를 눌러도 검은 창이 안 뜨고, 당근 브라우저는 자동으로 최소화되며, **서버 대시보드에서 눌러도** 로컬이 대신 수집합니다. 로컬 PC 자동 스케줄로 사실상 무인 갱신도 됩니다. 화면 UX도 개선했습니다.

## 새로워진 점 (요약)

| 구분 | 내용 |
|------|------|
| 🔇 **마케팅 동기화 = 백그라운드** | 검은 cmd 콘솔 창 없이 숨김 실행(로그는 파일). 당근·네이버는 로그인 세션상 브라우저만 잠깐/상주 |
| 🪟 **당근 브라우저 자동 최소화** | 로그인돼 있으면 CDP로 창을 바로 최소화, 세션 만료 시에만 표시 |
| 🖥️ **서버에서도 동기화 버튼 동작** | 서버에서 누르면 '수집 요청' 플래그 → 로컬 당근 데몬이 30초 폴링으로 감지 → 당근+네이버 수집 → 서버 업로드(원격 트리거) |
| ⏰ **로컬 PC 자동 스케줄** | 네이버 매일 2회(작업 스케줄러)·당근 로그인 시 상주(시작프로그램) → 무인 자동 갱신 |
| 🔁 **새로고침 시 현재 메뉴 유지** | 선택 메뉴를 localStorage 저장 → F5 해도 보던 페이지 유지 |
| ✅ **마케팅 완료 시 자동 새로고침** | 당근·네이버 데이터 갱신을 감시해 반영되면 자동 새로고침(시차 나는 두 소스 각각 감지, ~4분) |

## 마케팅 수집 방식(요약)
- **로컬 대시보드**에서 동기화 → 로컬이 즉시 백그라운드 수집.
- **서버 대시보드**에서 동기화 → 서버가 요청 표시 → 로컬 당근 데몬이 감지·수집·업로드(로컬 PC 켜져 있어야 함, ~1분).
- **자동**: 네이버 매일 2회 + 당근 상주(30분). 로컬 PC 켜져 있으면 무인.
