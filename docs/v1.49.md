## 개요
가맹점·단말기 수치가 "등록 기준"과 "거래 기준"이 섞여 상식(단말기 ≥ 가맹점)에 안 맞던 문제를 바로잡고, 기준을 **최근 30일 결제(운영)** 와 **역대 실제 거래**로 통일했습니다. 또 아무도없개 프랜차이즈 매출을 코밴 리포트에서 **가맹점명 필터로 매월 자동 집계**합니다.

핵심:
1. **기준 통일** — 가맹점 사용현황의 코밴·다우·전체 카드를 모두 **사업자번호(가맹점)·최근 30일 결제** 기준으로. 단말기 수(참고)만 하단에 표기.
2. **누적은 거래 기준** — 등록 단말기 전체 목록을 주는 리포트가 없어(코밴), '역대 실제 거래한' 단말기·가맹점으로 집계 → 단말기(2,130) ≥ 가맹점(1,528) 상식 성립.
3. **아무도없개 매출 자동화** — 2026-06부터 코밴 '가맹점별 매출금액 구간집계'에서 자동 수집(가맹점명 변형까지 포함). 그 전(24.11~26.5)은 수기 정적 데이터와 자동 병합.

## 새로워진 점 (요약)

| 구분 | 내용 |

| 🆕 **누적 가맹점수·단말기수** | 카드 2종 신설 — **역대 실제 거래** 기준(거래 단말기 2,130 · 거래 가맹점 1,528, KICC 제외). 코밴 TermCnt '일자별'(합계뷰는 280 캡) + 다우 정산 월별 union |
| 🆕 **아무도없개 매출 자동수집** | 매출현황 맨 아래 섹션. 코밴 `fixedrate`에서 가맹점명 `[아이]무도 없개`(오타·띄어쓰기 변형 포함) 필터로 월별 건수·금액. 연도 필터·금액/건수 토글 |

## 동작·데이터 메모
- **누적 단말기/가맹점**: 코밴은 거래(TermCnt 일자별) 기준 월별 union(과거월 캐시·현재월만 재조회), 다우는 정산내역 월별 union. 등록만 하고 거래 안 한 곳은 제외되므로 '실제 영업 규모'에 가깝습니다.
- **아무도없개 매출**: 대리점 **A25700(다인)+A25701(아무도없개) 둘 다 조회 후 사업자번호로 union** → 대리점 이동에도 안전(6월은 A25700에 있었음). 합계건수=끝-3열, 합계금액=끝-2열. 1천원 이하 거래는 리포트 정의상 합계에서 제외됩니다.
- **자동 갱신**: 신규 스크래퍼 3종을 매일 08:00 자동수집·해당 화면 '데이터 동기화' 스코프에 등록. 월별 캐시라 매일은 증분(빠름).
