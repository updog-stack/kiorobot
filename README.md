# 다인아이앤씨 통합 대시보드 (ERP) — v0.1

회사의 여러 시스템(노션 · VAN사 · 채널톡 · 국세청)에 흩어진 데이터를 **한 화면에서 실시간으로 보는 사내 통합 대시보드**입니다.
이 저장소의 `v0.1`은 핵심 운영 기능을 갖춘 **프로토타입**입니다.

> 자세한 문서는 [`dashboard/README.md`](dashboard/README.md), 운영 서버 배포는 [`dashboard/HOSTING.md`](dashboard/HOSTING.md)를 참고하세요.

## 기술 스택

이 프로젝트는 **① 화면(프론트엔드) ② 중계 서버(BFF) ③ 자동 수집기(스크래퍼)** 세 부분으로 구성됩니다.

| 구분 | 도구 | 설명 |
|------|------|------|
| 화면 | **React + TypeScript + Vite** | 컴포넌트 기반 UI, 타입 안정성, 빠른 개발 서버/빌드 |
| 중계 서버(BFF) | **Node.js + Express** | API 키를 서버에만 보관하고 외부 API를 대신 호출 |
| 자동 수집 | **Playwright** | 자동 로그인 · 클릭 · 다운로드로 VAN사 데이터 수집 |
| 데이터 처리 | **xlsx(SheetJS), imapflow** | 엑셀 파싱, Gmail 인증번호 자동 읽기(2차 인증) |
| 외부 연동 | **노션 / 채널톡 / 국세청 / Claude API** | 각 사 공식 API |
| 자동 실행 | **윈도우 작업 스케줄러** | 매일 정해진 시각 수집기 자동 실행 |

### 왜 중계 서버(BFF)가 필요한가
노션 · 채널톡 등의 **API 키는 브라우저에 직접 넣으면 안 됩니다**(누구나 볼 수 있게 됨).
그래서 화면과 외부 서비스 사이에 작은 서버(BFF, Backend-for-Frontend)를 두고 **키는 서버만** 보관합니다.
화면 → 우리 서버 `/api/...` 호출 → 서버가 키로 외부 API 호출 → 가공해 화면에 전달.

```
erp/
├─ README.md                 ← (이 파일) 프로젝트 개요
├─ master.md                 ← 비전 · 기획 문서
└─ dashboard/                ← 실제 대시보드 앱
   ├─ src/                   프론트엔드(React)
   │  ├─ components/         화면 컴포넌트
   │  └─ lib/                데이터 로직
   ├─ server/               중계 서버(BFF) + 스크래퍼
   ├─ README.md             대시보드 상세 문서
   └─ HOSTING.md            운영 서버 배포 가이드
```

## 실행 방법 (개발용)

> Node.js(LTS) 설치 필요. API 연동 기능은 `dashboard/.env` 설정이 있어야 동작합니다(`.env.example` 참고).

```bash
cd dashboard
npm install
npm run dev                          # 화면 (http://localhost:5173)
node server/notion-sales-bff.mjs     # 중계 서버 API (http://localhost:8787)
```

브라우저에서 `http://localhost:5173` 접속.
