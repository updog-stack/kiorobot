# 운영 서버(호스팅) 가이드

프로토타입을 **사내 PC가 아닌 외부 서버**에서 24시간 운영하기 위한 정리입니다.
(업체 비교 · 비용 · 배포 방법 · 운영 시 주의점)

> 비용은 2026년 기준 **대략치**이며 사양·환율·프로모션에 따라 달라집니다. 신청 전 각 사 요금표를 확인하세요.

---

## 1. 이 앱이 서버에 요구하는 것

이 앱은 단순 정적 사이트가 아니라 **항상 켜져 있어야 하는 서버 + 자동 수집기**가 필요합니다.

| 요구 | 이유 |
|------|------|
| **상시 가동 리눅스 서버** (VPS/클라우드 1대) | 중계 서버(BFF)가 계속 떠 있어야 함 |
| **Node.js 20+** | 서버·수집기 실행 |
| **헤드리스 크롬(Playwright)** | 코밴·다우데이타 자동 로그인/스크래핑 |
| **cron(예약 실행)** | 매일 08:00 수집(윈도우 작업 스케줄러 대체) |
| **디스크 약간** | 수집 데이터(JSON), 추후 DB |
| **도메인 + HTTPS** | 안전한 접속 주소 |
| **접근 제어** | ⚠️ 현재 로그인이 없어 **반드시** 사내 IP 제한 또는 로그인 추가 |

권장 최소 사양: **2 vCPU / 4GB RAM / 50GB SSD** (5인 사내용으로 충분). Playwright 때문에 1GB RAM은 빠듯합니다.

| 업체 | 유형 | 사양 예시 | 월 비용(대략) | 장점 | 단점 |
|------|------|-----------|---------------|------|------|
| **네이버 클라우드(NCP)** | 국내 클라우드 | 2vCPU/4GB | **₩4만~9만** | 한국어 콘솔·지원, 국내 데이터, KRW 결제 | 글로벌 대비 다소 비쌈 |
| **카페24 / 가비아 VPS** | 국내 호스팅 | 2~4GB | **₩1만~5만** | 저렴, 한국어, 전화 지원 | 콘솔/기능 단순 |
| **AWS Lightsail (서울)** | 글로벌 간편 클라우드 | 2GB($10)/4GB($20) | **₩1.4만~3만** | 저렴·정액·서울 리전, 확장 쉬움 | 콘솔 영어, USD 결제 |
| **DigitalOcean / Vultr / Linode** | 글로벌 VPS | 2~4GB | **₩1만~3만** | 저렴·간단, 개발자 친화 | 한국어 지원 없음 |
| **Render / Railway / Fly.io** | PaaS(관리형) | - | **$7~$25 (₩1만~3.5만)** | 서버 관리 거의 불필요, 배포 쉬움 | 스크래퍼(Playwright)·cron·영구디스크에 추가 설정/비용, 제약 있음 |

### 추천
- **가장 안정/국내 지원 우선** → **네이버 클라우드(NCP)** 또는 **가비아·카페24 VPS** (한국어·KRW·국내 리전. 사업자번호·CS 데이터라 국내 보관이 깔끔)
- **가성비/확장성 우선** → **AWS Lightsail 서울 리전 (4GB, 약 ₩3만/월)**
- **서버 운영을 최대한 안 하고 싶다** → Render 등 PaaS. 단 스크래핑·예약작업 때문에 이 앱엔 **VPS가 더 단순**합니다.

> 결론(권장): **NCP 또는 Lightsail(서울)에 VPS 1대 + 도메인**. 전부 합쳐 **월 ₩3만~7만** 수준.

## 3. 권장 구성

```
[도메인 erp.daininc.kr] ──HTTPS──> [Nginx 또는 Caddy]
                                      ├─  /         → React 빌드(dist) 정적 제공
                                      └─  /api/*     → 중계 서버(BFF, Node 8787)로 전달
[PM2/systemd] : BFF를 항상 실행 + 죽으면 자동 재시작
[cron]        : 매일 08:00  node server/daily-collect.mjs
[접근제어]    : 사내 IP만 허용  또는  로그인(인증) 추가
```

## 4. 배포 절차 (Ubuntu 기준, 한 번만)

```bash
# 0) 서버 생성: Ubuntu 22.04 LTS, 2vCPU/4GB. SSH로 접속.

# 1) Node 설치 (nvm)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
. ~/.nvm/nvm.sh && nvm install 22

# 2) 코드 받기 (Private 저장소 → GitHub PAT 또는 deploy key 사용)
git clone https://github.com/<계정>/dain-erp-dashboard.git
cd dain-erp-dashboard

# 3) 설치 + 브라우저(리눅스 의존성 포함)
npm install
npx playwright install --with-deps chromium

# 4) 비밀키 작성 (서버에 직접, 깃엔 없음) — 운영용으로 재발급한 키 사용
cp .env.example .env && nano .env

# 5) 프론트 빌드
npm run build      # dist/ 생성

# 6) 중계 서버 상시 실행 (PM2)
npm i -g pm2
pm2 start server/notion-sales-bff.mjs --name erp-bff
pm2 save && pm2 startup     # 재부팅 후 자동 실행

# 7) 매일 08:00 수집 (cron)
crontab -e
#   아래 한 줄 추가 (경로는 실제 경로로):
#   0 8 * * *  cd /home/ubuntu/dain-erp-dashboard && /home/ubuntu/.nvm/versions/node/v22*/bin/node server/daily-collect.mjs >> /home/ubuntu/collect.log 2>&1
```

### 정적 화면 + /api 연결 (둘 중 하나)
- **방법 A — Nginx 리버스 프록시(권장):** `/` 는 `dist/` 정적 제공, `/api/` 는 `localhost:8787` 로 프록시. + `certbot` 으로 무료 HTTPS.
- **방법 B — Caddy:** 설정 한 줄로 자동 HTTPS + 프록시(가장 간단).
- **방법 C — BFF가 직접 서빙:** `notion-sales-bff.mjs` 에 `app.use(express.static("dist"))` 한 줄 추가하면 8787 한 포트로 화면+API 모두 제공(가장 코드 단순, 이 방법을 원하면 제가 추가해 드립니다).

## 5. 운영 시 꼭 챙길 것

1. **접근 제어(가장 중요)** — 현재 대시보드에 **로그인이 없습니다.** 인터넷에 그대로 열면 누구나 봅니다. → 사내 고정 IP만 허용(방화벽/Nginx allow), VPN, 또는 간단 로그인(Basic Auth/구글 OAuth) 추가 필요. *원하면 로그인 기능을 붙여 드립니다.*
2. **비밀키 관리** — `.env`는 서버에만. 운영 키는 프로토타입 때 노출분과 다른 **새 키**로.
3. **프로세스 자동 복구** — PM2/systemd로 죽으면 재시작.
4. **백업** — `server/data/*.json`(수집 결과)와 `.env`를 주기 백업. (스크래퍼가 재생성 가능하지만 안전하게)
5. **2차 인증(다우데이타)** — Gmail IMAP은 서버에서도 동작. 단 채널톡/VAN사가 새 기기(서버 IP) 로그인 시 추가 인증을 요구할 수 있으니 최초 1회 확인 필요.
6. **로그/모니터링** — 수집 실패 시 알림(예: 슬랙)로 받으면 운영이 편합니다.
7. **외부 사이트 변경 대응** — 코밴/다우데이타 화면이 바뀌면 스크래퍼 수정이 필요(유지보수 항목).

## 6. 데이터 저장: JSON → DB (운영 단계 권고)

- 지금은 수집 결과를 **JSON 파일**(`server/data/`)에 저장합니다. 프로토타입엔 충분하지만, 데이터가 쌓이고 여러 명이 동시에 쓰면 **데이터베이스(PostgreSQL)** 로 옮기는 걸 권합니다.
- 또한 master.md 원칙대로 일부 데이터(매출 등)는 **노션을 원본**으로 두는 구조도 고려.
- 이 전환은 별도 작업으로, 운영 안정화 후 진행하면 됩니다.
