# 일정 탭 ↔ 구글캘린더 양방향 연동 설정 (대표님용)

ERP에서 일정을 **생성·수정·삭제**하고 구글캘린더와 **거의 실시간(45초)** 으로 동기화하려면
아래 **서비스 계정** 설정이 필요합니다. (한 번만, 약 10분)

> 캘린더 ID는 미리 채워뒀습니다(`GOOGLE_CALENDARS`). 서비스 계정 키 2개만 넣으면 됩니다.

전체 순서
1. 구글 클라우드 프로젝트 만들기
2. 캘린더 API 켜기
3. 서비스 계정 + 키(JSON) 내려받기
4. **두 캘린더를 서비스 계정에 '변경 권한'으로 공유** ⭐
5. `.env` 에 키 2개 넣기
6. 서버 재시작

---

## 1. 구글 클라우드 프로젝트
1. https://console.cloud.google.com → 회사 구글 계정 로그인
2. 맨 위 프로젝트 선택 → **[새 프로젝트]** → 이름 `dain-dashboard` → 만들기 → 선택
   (결제 카드 불필요·무료)

## 2. 캘린더 API 켜기
- ☰ → **API 및 서비스 → 라이브러리** → `Google Calendar API` 검색 → **[사용]**

## 3. 서비스 계정 + 키(JSON)
1. ☰ → **API 및 서비스 → 사용자 인증 정보**
2. **[+ 사용자 인증 정보 만들기] → 서비스 계정** → 이름 `dashboard-calendar` → 만들기 → 완료
3. 만든 서비스 계정 클릭 → **[키] 탭 → [키 추가] → 새 키 만들기 → JSON → 만들기**
   → JSON 파일이 다운로드됩니다. (비밀 키 포함, 잘 보관)
4. 서비스 계정 **이메일** 복사 (`...@....iam.gserviceaccount.com`)

## 4. 두 캘린더를 서비스 계정에 공유 ⭐ 가장 중요
**'다인 설치일정'** 과 **'설치외 방문일정'** 두 캘린더 모두 아래를 반복:

1. https://calendar.google.com → 캘린더 위 **⋮ → 설정 및 공유**
2. **'특정 사용자 또는 그룹과 공유' → [사용자 추가]** → 위 서비스 계정 이메일 붙여넣기
3. 권한을 **'일정 변경 및 공유 관리'** 로 선택 ← (보기 아님! 쓰기까지 하려면 변경 권한)
4. 보내기

> ⚠️ 캘린더 소유자가 외부(다른 회사)면 '변경 권한' 부여가 막힐 수 있습니다.
> 그 경우 캘린더 주인에게 위 이메일을 '변경 및 관리' 권한으로 공유해달라고 요청하세요.

## 5. `.env` 에 키 2개 넣기
`dashboard` 폴더의 **`.env`** 를 메모장으로 열고:

```
GOOGLE_SA_EMAIL=서비스계정이메일
GOOGLE_SA_PRIVATE_KEY=JSON의 private_key 값 전체(중간 \n 그대로, 한 줄)
```
- **GOOGLE_SA_EMAIL** : 3-4의 이메일
- **GOOGLE_SA_PRIVATE_KEY** : 다운로드한 JSON 파일을 메모장으로 열어
  `"private_key": "..."` 의 **큰따옴표 안 내용 전체**를 복사해 붙여넣기
  (예: `-----BEGIN PRIVATE KEY-----\nMIIE...\n-----END PRIVATE KEY-----\n`, `\n` 은 지우지 말 것)
- `GOOGLE_CALENDARS` 는 **이미 채워져 있습니다** (두 캘린더 ID). 그대로 두세요.

## 6. 서버 재시작
1. **'데이터 API (BFF)' 검은 창**을 닫기
2. **`실행.bat`** 다시 더블클릭
3. 일정 탭 새로고침 → 우측 상단 **'+ 일정 추가'** 버튼이 보이면 양방향 모드 성공 🎉

---

## 사용법 (설정 후)
- **추가**: '+ 일정 추가' 버튼, 또는 달력의 **빈 날짜 클릭**
- **상세/수정/삭제**: 일정 클릭 → 모달에서 **수정 / 삭제**
- **동기화**: 구글캘린더에서 바꾼 내용은 ERP가 **45초 안에** 자동 반영. ERP에서 바꾼 건 구글에 **즉시** 반영.

## 문제 해결
| 증상 | 해결 |
|------|------|
| '+ 일정 추가' 버튼이 없음 / '읽기 전용' 표시 | SA 키 미입력 또는 BFF 재시작 안 함 → 5·6단계 |
| 저장 시 "권한 오류 / 403" | 4번에서 '보기'로만 공유함 → '변경 및 관리'로 다시 |
| "404 Not Found" | 캘린더 공유 누락, 또는 GOOGLE_CALENDARS ID 불일치 |
| "구글 인증 실패" | PRIVATE_KEY 잘림 / `\n` 지움 → JSON에서 다시 복사 |

> 막히면 JSON 파일을 담당자에게 전달하면 대신 넣어드립니다.

## (참고) 읽기 전용만 원할 때 — iCal 방식
양방향이 필요 없고 보기만 하려면, 서비스 계정 없이 `.env` 의
`GOOGLE_CALENDAR_ICS_URL` 에 캘린더의 'iCal 비공개 주소'만 넣어도 됩니다.
단 이 경우 ERP에서 편집은 불가하고, 구글 변경 반영이 수 시간 지연될 수 있습니다.
(서비스 계정 키가 설정되면 그쪽이 우선합니다.)
